# 快速开始 - 多项目管理系统

## 🎯 核心改进

系统现在支持**多项目管理**，每个小说拥有独立的：
- ✅ 配置文件
- ✅ 数据库（进度保存）
- ✅ 章节文件目录
- ✅ 世界观数据

**不再需要每次写新小说都删除旧数据！**

## 🚀 快速使用

### 方法1：使用novel.sh（推荐）

```bash
# 创建新项目
./novel.sh new

# 生成章节
./novel.sh generate

# 管理所有项目
./novel.sh projects

# 查看系统状态
./novel.sh status
```

### 方法2：直接使用Python脚本

```bash
# 创建新项目
python3 configure_novel.py

# 生成章节
python3 main.py

# 管理项目
python3 manage_projects.py
```

## 📖 典型场景

### 场景1：开始第一个项目

```bash
./novel.sh new
# 输入小说信息（标题、类型、章节数等）

./novel.sh generate
# 开始生成章节，可随时Ctrl+C中断，进度自动保存
```

### 场景2：同时写两本小说

```bash
# 第一本
./novel.sh new
# 标题：末日真理
./novel.sh generate
# 生成10章...

# 第二本
./novel.sh new
# 标题：剑道独尊
./novel.sh generate
# 生成20章...

# 切换回第一本
./novel.sh projects
# 选择"切换项目" → "末日真理"

./novel.sh generate
# 自动从第11章继续
```

### 场景3：查看所有项目

```bash
./novel.sh status
```

## 🎨 novel.sh 命令总览

| 命令 | 简写 | 说明 |
|------|------|------|
| `new` | `n` | 创建新项目 |
| `generate` | `g` | 生成章节 |
| `projects` | `p` | 管理项目（切换/删除/查看） |
| `status` | `s` | 查看系统和项目状态 |
| `help` | `h` | 显示帮助 |

## 📂 文件位置

所有项目数据存储在：
```
/project/novel/projects/<项目名>/
├── config.yaml      # 项目配置
├── state.db         # 生成进度
├── manuscript/      # 章节文件
└── bible/           # 世界观数据
```

## 💡 常见问题

**Q: 如何删除不需要的项目？**
```bash
./novel.sh projects
# 选择"删除项目" → 选择要删除的项目 → 输入"yes"确认
```

**Q: 如何知道当前在哪个项目？**
```bash
./novel.sh status
# 当前项目会有"👉"标记
```

**Q: 可以修改已创建的项目配置吗？**
可以，编辑项目目录下的 `config.yaml` 文件

**Q: 中断生成后数据会丢失吗？**
不会！进度自动保存，下次运行会提示从断点继续
